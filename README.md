# つぶやきメモ (memo-pad)

> 今を記録するメモ帳です。今の心の中にある何かに名前をつけていきましょう。あなたの言葉は、このブラウザの中だけに、しずかに残ります。

---

## ① 課題名

**つぶやきメモ**

## ② 課題内容（どんな作品か）

「いまの気持ちは？」── 心の中にあるものに名前をつけ、「何があったかも残しておく」、X (旧 Twitter) 風のタイムライン型メモパッドです。保存先はブラウザの `localStorage` だけ。外部サーバには一切送られません。

- **タイムライン型メモパッド**：投稿フォーム + 過去ログの2構成。新しい投稿が常に上に並ぶ
- **主従の入力フォーム**：
  - **いまの気持ちは？（必須）** ── 縦に広めの textarea で、心の中にある何かに名前をつける（例: ホッとした、なんだかうれしい、モヤモヤ）
  - **何があったかも残しておく（任意）** ── 出来事をそのまま残しておく場所（例: 朝のミーティングで思いがけず褒められた。）
- **読込時 autofocus**：ページに入ると自動で「いまの気持ちは？」欄にカーソルが入り、青のフォーカスリングで「ここに書く」が一目で分かる
- 授業で学んだ `localStorage` をベースに、課題で要求された **1件削除** と **1件更新** を追加した発展版
- さらに **検索フィルタ** と **作成日時表示**、**新しい順タイムライン** を自分の工夫として実装
- 編集はモーダルを開かず、ひとこと・本文を **その場でクリックして直接編集** (`contenteditable`) できる

## ③ アプリのデプロイURL

GitHub Pages: <https://hiro-yamazaki.github.io/memo-pad/>

## ④ アプリのログイン用IDまたはPassword

- ID: なし

## ⑤ 工夫した点・こだわった点

### 主な工夫ポイント

- **X風タイムラインUI**：投稿エリアを上部に固定し、過去ログがその下に新しい順で並ぶ。各ポストは **アバター丸 → 日時 → いまの気持ち → 本文 → アクションバー(返信/リポスト/いいね/ブックマーク)** の構成で、本物のXのヘッダー(アバター横に日時) を再現
- **主従の入力フォーム**：「いまの気持ちは？(必須)」を 22px 太字・縦96px 以上の大きな textarea でメインに据え、「何があったかも残しておく(任意)」を 15px の控えめなテキストエリアで脇に置く構造。ラベルに **必須 / 任意** バッジを併記し、初心者でも「最低限ひとことだけ書けば投稿できる」ことが一目で分かる。読込時 `autofocus` で気持ち欄にすぐ書き始められる
- **保存後の視覚フィードバック**：「つぶやく」を押すと入力欄をクリアして `#title` にフォーカスを戻し、`.just-saved` クラスで **青リングが0.7秒脈動** (`title-save-pulse`)。「保存できた → ここに次を書ける」を視覚で伝える
- **アクションバーに小演出**：
  - **リポスト** にカーソルを乗せると、そのポストが `#list` の先頭に `prependTo` で移動 + 緑のグローが0.8秒フェード (`repost-glow`)
  - **いいね** クリックで ♡ → ♥ にトグル、ハートが軽くバウンス (`like-burst`) し、隣に「100」がポップ表示 (`count-pop`)。再クリックで元に戻る
- **白背景 × X青の差し色**：背景 `#ffffff` / 本文 `#0f1419` / 補助 `#536471` / アクセント `#1d9bf0` の高コントラスト配色（WCAG AAA 余裕、コントラスト比16.7:1）で可読性を最優先。「つぶやく」ボタンとフォーカス枠が青く、視線が自然に誘導される
- **新しいものが上のタイムライン**：`createdAt` を JSON に同梱し、初期復元時は降順ソート、新規保存時は `append` ではなく **`prepend`** で先頭に挿入
- **インライン編集**：いまの気持ち・本文をクリックするとその場で編集できる (`contenteditable="true"` + blur で保存)。専用の編集モーダルを作らず、最小UIで完結
- **見出し(キー)変更にも対応**：`localStorage` のキー = いまの気持ち という設計なので、変更時は「旧キーを `removeItem` → 新キーで `setItem`」の2手順で書き換え。重複の衝突もチェック
- **作成日時の保持**：編集や上書き保存をしても作成日時は維持する
- **検索フィルタ**：上部の検索欄に文字を入れると `input` イベントで即座に絞り込み。気持ち・本文どちらにヒットしてもOKで、大文字小文字を区別しない
- **XSS への配慮**：jQuery の `.text()` のみを使い、ユーザー入力を `.html()` で埋め込まない
- **コメント手厚め**：後で自分が読み返したときに学習目的が思い出せるよう、「作る順番」「使う技術」「学習意図 (なぜこのコードか)」を `js/script.js` のセクションごとに記載

### 動作確認方法

1. `index.html` をブラウザで開く（または上記デプロイURL）
2. ページに入るとカーソルが自動で「**いまの気持ちは？**」欄に入る。
   - **いまの気持ちは？（必須）**: 心の中にある何かに名前をつける（例: ホッとした、なんだかうれしい、モヤモヤ）
   - **何があったかも残しておく（任意）**: 出来事をそのまま（例: 朝のミーティングで思いがけず褒められた。）
   を書いて「**つぶやく**」をクリック → タイムラインの一番上に追加される
3. ブラウザを閉じて再度開いても残っている (`localStorage` で保存されているため)
4. 各記録のひとこと or 本文を **クリック → その場で編集 → 別の場所をクリック (blur)** で上書き保存
5. 各記録の右上の `×` ボタンで1件削除、「すべて消す」で全消去
6. 上部の検索欄に文字を打つと該当記録のみ表示される
7. 各ポストの下のアクションバー：
   - **返信 / ブックマーク** ── 見た目だけの装飾
   - **リポスト** ── アイコンに **カーソルを乗せる** とそのポストが最上段に移動 + 緑グロー
   - **いいね** ── クリックで ♡ → ♥ にトグル、隣に「100」がポップ表示（再クリックで戻る）

### デザインメモ

- 配色は **白 + X青1色** に絞り、可読性最優先 (`#ffffff` / `#0f1419` / `#1d9bf0`)。本文の `#0f1419` は背景 `#fff` に対してコントラスト比 約16.7:1 で WCAG AAA 余裕
- 「つぶやく」ボタンのみ X 青、それ以外はモノクロ ── 視線が自然に「投稿」へ誘導される
- ヘッダーは `つぶやきメモ @tubuyaki_memo` の2要素を baseline 揃えで横並びにし、X のブランドバー風に。`position: sticky` + `backdrop-filter: blur(12px)` でスクロール時も常駐
- 入力欄は **主従構造** にデザイン：
  - いまの気持ちは？(必須): 22px 太字 textarea (縦96px〜)、枠付き、青フォーカスリング(4px ぼかし)、必須バッジ(赤)、`autofocus` + `caret-color` で読込時から書き始められる状態を視覚化
  - 何があったかも残しておく(任意): 15px、線の細い枠、任意バッジ(灰) で控えめに
- ボタン配置は左から `[つぶやく][すべて消す]`。主要アクションを左に置き、視線の流れに合わせる
- 各ポストは **アバター丸(44px) + 日時 + いまの気持ち(17px太字) + 本文 + アクションバー** の縦並び。`grid-template-columns: 44px 1fr` で「左にアバター、右に縦積み」を実現
- フォントは system-ui (sans-serif) に統一、本文行間 1.5 で長い文章も読みやすく
- タイムラインはカード型ではなく上下ボーダーのみで区切り、X の連続フィードに寄せた

### localStorage で扱った API 一覧（学習メモ）

授業 (PDF p.21) で出てきた API と、このアプリでの使いどころ。

| API | 説明 | このアプリでの使用箇所 |
| --- | --- | --- |
| `localStorage.setItem(key, value)` | データ登録または更新 (上書き) | 「つぶやく」時 / インライン編集の保存時 |
| `localStorage.getItem(key)` | キーから値を取得 | リロード復元時 / 編集前の元データ取得 |
| `localStorage.removeItem(key)` | 1件削除 | × ボタン / 見出し変更時の旧キー削除 |
| `localStorage.clear()` | 全件削除 | 「すべて消す」ボタン |
| `localStorage.length` | 保存件数 | リロード復元時の for ループ条件 |
| `localStorage.key(i)` | i 番目のキー名 | リロード復元時のキー取得 |

### なぜ JSON で包んで保存しているか

授業のサンプルは `setItem(title, body)` と本文をそのまま値にしていますが、
このアプリでは **本文 + 作成日時** を一緒に保存したいので、値を JSON 文字列にしています。

```js
// 保存時
localStorage.setItem(title, JSON.stringify({ body: body, createdAt: createdAt }));

// 取り出し時
const obj = JSON.parse(localStorage.getItem(title));
// obj.body, obj.createdAt が使える
```

`localStorage` は **文字列しか保存できない** ので、構造化データを扱うにはこの一手間が必要。
これが PDF p.8 で言われていた「`localStorage` のデメリット」の1つで、
DB なら型 (text, datetime, ...) を素直に扱える ── という流れに繋がっています。

## ⑥ 難しかった点・次回トライしたいこと

- **`localStorage` のキー rename ができないこと**：UPDATE 文一発で済む DB と違い、`localStorage` は「キー名を変える」API がない。見出しを編集するときは「旧キーを消す → 新キーで作り直す」しかなく、ここで授業 (PDF p.8) で言われていた「DB が必要な理由」を体感した
- **キーの並び順がブラウザ依存**：`localStorage.key(i)` は保存順を保証しない。タイムラインを「新しいものが上」にするために、自前で `createdAt` を持って降順ソートする必要があった。DB なら `ORDER BY` で一発
- **次回トライしたいこと**
  - タグ機能（複数のキーをまとめて検索）
  - 感情(見出し)別の集計グラフ
  - Markdown プレビュー
  - エクスポート（`.json` でダウンロード／インポート）
  - 投稿時のキーボードショートカット (Cmd+Enter)
  - 想定モニター10名のレビューで★が付いた改善（日時の相対表記、保存完了トースト、SVGアイコン化、編集ヒント、集計ビュー など）

## ⑦ フリー項目（感想、シェアしたいこと等）

### 学んだことのまとめ

- **`localStorage` は「文字列のキー → 文字列の値」しか持てない**。複数の情報をまとめたいときは `JSON.stringify` / `JSON.parse` で包む必要がある。これが地味に面倒で、確かに DB が欲しくなる
- **イベント委譲 (`$("#list").on("click", ".memo-delete", ...)`)** を初めて意識して使った。動的に `prepend` した要素にもイベントが効く仕組み
- **`contenteditable`** は1属性追加するだけで要素を編集可能にできる強力な仕組み。値の取得は `<textarea>` の `val()` ではなく `.text()` を使う
- **デザインの一貫性**：白背景に1色だけアクセント(X青)を置くと、視線が自然に誘導されて UI が静かになる。X や Linear が学ばせてくれること

### つくった背景・感想

授業の Memo Pad 課題を出発点に、「ただの保存・表示」で終わらせず **「心の中にある何かに名前をつける場所」** という小さなテーマを置いて、X (旧 Twitter) 風のタイムライン体験に寄せてみました。

特に大事にしたのは、**「最低限ひとことだけでOK」と思える主従の入力フォーム** と、**「ここに書く」が一目で分かる autofocus + 青リング** の2点で、「メモを残す」行為自体のハードルを下げることにこだわっています。

実体験のユーザーレビューは集められない代わりに、AIで多視点ペルソナ10名を立て、デザイン判断の盲点を点検するという「自己レビューのプロセス」を試したのも今回の挑戦です（README の ⑤ に掲載）。改善示唆として ★ がついた項目は次回のイテレーション候補として残してあります。
